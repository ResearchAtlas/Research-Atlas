## How to work a task doc

- Tasks are grouped by theme (e.g. "wiring", "corpus", "acceptance").
- Each task has a checkbox. Flip to `[x]` when actually done, not
  when drafted.
- If a task turns out to be wrong or obsolete, strike it through and
  add a dated note. Do not silently delete.
- Add tasks you discover in-flight. The checklist is not a contract;
  it's a running record of what we thought we needed to do.
- When a phase ends, its task doc becomes history. Do not delete —
  future contributors may want to see how the phase played out.
